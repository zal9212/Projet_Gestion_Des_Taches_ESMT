App accounts — User & Profile
Modèles User & Profile

Objectif : Créer un système d'utilisateurs avec rôles et profils.

Instructions : Dans accounts/models.py, créez une classe User (héritant de AbstractUser) avec un champ role (choix: etudiant/professeur). Créez ensuite une classe Profile liée au User par un OneToOneField pour l'avatar et la bio.

Inspiration : accounts/models.py
from django.contrib.auth.models import AbstractUser
from django.db import models

class User(AbstractUser):
    ROLE_CHOICES = [('etudiant', 'Étudiant'), ('professeur', 'Professeur')]
    role = models.CharField(max_length=20, choices=ROLE_CHOICES, default='etudiant')
    email = models.EmailField(unique=True)

class Profile(models.Model):
    user = models.OneToOneField(User, on_delete=models.CASCADE, related_name='profile')
    avatar = models.ImageField(upload_to='avatars/', blank=True, null=True)
    bio = models.TextField(blank=True)
Critique
Explication : Pourquoi un Custom User ?

Par défaut, Django n'a pas de champ "rôle". En utilisant AbstractUser, vous gardez la flexibilité d'ajouter vos propres champs (comme role) dès le départ, ce qui est une bonne pratique indispensable pour toute application sérieuse.

Modèle Task (Auto-completion)

Ajoutez une logique automatique pour la date de complétion.

tasks/models.py
from django.utils import timezone

class Task(models.Model):
    project = models.ForeignKey(Project, on_delete=models.CASCADE)
    status = models.CharField(max_length=20, default='todo')
    completed_at = models.DateTimeField(null=True, blank=True)

    def save(self, *args, **kwargs):
        if self.status == 'done' and not self.completed_at:
            self.completed_at = timezone.now()
        super().save(*args, **kwargs)

Concept — JWT (JSON Web Token)

Après login, le serveur génère 2 tokens : access (valide 60 min, envoyé à chaque requête) et refresh (7 jours, pour renouveler l'access). Le client l'envoie dans le header HTTP : Authorization: Bearer eyJ.... Avantage : le serveur n'a pas besoin de stocker de session — il vérifie juste la signature du token.

Créer le Profil automatiquement (Signal)

Utilisez un signal post_save pour garantir que chaque utilisateur possède un profil dès sa création.

accounts/signals.py
from django.db.models.signals import post_save
from django.dispatch import receiver
from .models import User, Profile

@receiver(post_save, sender=User)
def create_user_profile(sender, instance, created, **kwargs):
    if created:
        Profile.objects.create(user=instance)

Note : N'oubliez pas d'importer ce fichier dans accounts/apps.py dans la méthode ready().

from django.db.models import Q
from rest_framework import viewsets
from .models import Project
from .serializers import ProjectSerializer

class ProjectViewSet(viewsets.ModelViewSet):
    serializer_class = ProjectSerializer

    def get_queryset(self):
        user = self.request.user
        return Project.objects.filter(
            Q(owner=user) | Q(members__user=user)
        ).distinct()

Concept — CORS (Cross-Origin Resource Sharing)

Par défaut, un navigateur bloque toute requête vers un domaine différent (ex: React sur localhost:3000 ne peut pas appeler Django sur localhost:8000). django-cors-headers ajoute les headers HTTP Access-Control-Allow-Origin qui autorisent ces échanges. En dev : CORS_ALLOW_ALL_ORIGINS = True. En prod : restreindre à ton domaine.

Concept — useEffect + useState (chargement de données)

Règles Métier Critiques
Permissions & Rôles
· Seul l'owner peut ajouter/supprimer des tâches ou membres
· Un membre assigné peut uniquement modifier le statut
· Un étudiant ne peut pas assigner un professeur
· Non-membres : accès refusé (403)
Calcul des Primes (Professeurs)
· Taux 100% → 100 000 FCFA
· Taux ≥ 90% et < 100% → 30 000 FCFA
· Taux < 90% → Aucune prime
· Tâche "dans les délais" : completed_at ≤ due_date
